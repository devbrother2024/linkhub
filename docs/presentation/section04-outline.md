# SECTION 04. 수익화로 확장하기: 링크 단축 SaaS 서비스 LinkHub 개발 (국내 결제)

---

## 챕터 소개

이번 챕터에서는 **LinkHub 링크 단축 SaaS 서비스**를 처음부터 끝까지 개발합니다.

링크를 짧게 만들고, 사용자 인증을 구현하고, **Toss Payments**를 통한 국내 결제 시스템을 연동해 수익화까지 완성하는 실전 프로젝트입니다.

> **참고**: 해외 결제(Stripe 등) 연동은 다른 챕터에서 진행 예정입니다.

---

## 이번 챕터에서 여러분이 얻을 것

- **프론트엔드**: Next.js 기반 풀스택 개발 경험
- **백엔드**: API Route와 데이터베이스 ORM 활용
- **인증**: Better Auth를 통한 소셜 로그인 구현
- **국내 결제**: Toss Payments 연동으로 수익화 모델 구축 (원화 결제)
- **배포**: 프로덕션 환경 배포와 운영 경험
- **전체 흐름**: 기획부터 배포까지의 SaaS 개발 프로세스 이해

> **참고**: 해외 결제(Stripe 등) 연동은 다른 챕터에서 학습합니다.

## 이번 챕터의 차별점

**SECTION 03과 비교했을 때 새롭게 추가되는 부분**

### 1. 프레임워크 전환

- **Vite + React** → **Next.js**
- 서버 사이드 렌더링(SSR)과 API Route 활용
- 풀스택 개발 경험

### 2. 데이터베이스 스키마 관리

- **Supabase Database** (동일하게 사용)
- **ORM 도입**: Prisma 또는 Drizzle을 통한 스키마 관리
- 마이그레이션 파일 기반 스키마 버전 관리
- 타입 안전한 데이터베이스 쿼리 작성

### 3. 인증 솔루션

- **이메일 인증** → **Better Auth + 소셜 로그인**
- OAuth 플로우 이해
- 세션 관리와 보안 고려사항

### 4. 국내 결제 연동

- **새로 추가**: Toss Payments 연동 (원화 결제)
- 구독 모델 설계 (Free/Pro 플랜)
- 수익화 전략 구현
- **참고**: 해외 결제(Stripe 등)는 다른 챕터에서 다룹니다

### 5. 배포 플랫폼

- **Netlify** → **Vercel**
- 서버리스 함수와 환경 변수 관리
- 프로덕션 배포 프로세스

## 기술 스택 개요

### 프론트엔드 & 백엔드

- **Next.js**: React 기반 풀스택 프레임워크
- **TypeScript**: 타입 안정성 확보
- **Tailwind CSS**: 빠른 UI 개발

### 데이터베이스 & ORM

- **Supabase**: PostgreSQL 기반 클라우드 데이터베이스
- **Prisma / Drizzle**: 타입 안전한 ORM으로 스키마 관리
- 마이그레이션을 통한 체계적인 스키마 버전 관리

### 인증

- **Better Auth**: 모던 인증 솔루션
- **소셜 로그인**: 국내 주요 플랫폼 연동

### 결제

- **Toss Payments**: 국내 결제 대행사(PG), 원화 결제 지원
- **구독 모델**: Free/Pro 플랜 구조
- **참고**: 해외 결제 시스템(Stripe 등)은 다른 챕터에서 학습합니다

### 배포

- **Vercel**: Next.js 최적화 배포 플랫폼
- **GitHub**: 버전 관리 및 CI/CD 연동
